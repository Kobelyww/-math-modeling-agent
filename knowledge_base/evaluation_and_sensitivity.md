# 模型检验、灵敏度分析与评价方法

## 一、模型检验方法总览

### 检验维度
1. **正确性检验** — 模型结果是否正确（与真实值/标准答案对比）
2. **稳定性检验** — 小扰动下结果是否稳定
3. **鲁棒性检验** — 不同条件下模型是否仍可用
4. **敏感性检验** — 各参数对结果的影响程度
5. **合理性检验** — 结果是否与常识/理论一致

## 二、误差度量指标

### 回归/预测问题
| 指标 | 公式 | 特点 |
|------|------|------|
| MSE | (1/n)Σ(yi-ŷi)² | 放大大误差，量纲为平方 |
| RMSE | √MSE | 与原数据同量纲 |
| MAE | (1/n)Σ|yi-ŷi| | 对大误差不敏感，更稳健 |
| MAPE | (1/n)Σ|(yi-ŷi)/yi|×100% | 相对误差，适合比较不同量级 |
| R² | 1 - SS_res/SS_tot | 越大越好，≤1 |
| 调整R² | 1-(1-R²)(n-1)/(n-p-1) | 考虑变量个数p的修正 |

### 分类问题
| 指标 | 公式 | 适用场景 |
|------|------|----------|
| 准确率 | (TP+TN)/(TP+TN+FP+FN) | 类别平衡时 |
| 精确率 | TP/(TP+FP) | 关注误报代价 |
| 召回率 | TP/(TP+FN) | 关注漏报代价 |
| F1-score | 2×P×R/(P+R) | 精确率与召回率的调和平均 |
| AUC-ROC | — | 整体分类性能，对不平衡鲁棒 |

### 选择建议
- 数据有极端值 → 用MAE而非MSE
- 需要相对比较 → 用MAPE（但分母不能接近0）
- 类别不平衡 → 用F1/AUC而非准确率

## 三、交叉验证

### K-fold交叉验证
- **步骤**：数据随机分K份 → 依次用K-1份训练、1份验证 → K次结果取平均
- **K的选择**：通常取5或10，建议K=10（偏差-方差平衡）
- **分层K-fold**：每份保持原始类别比例（分类问题推荐）

### 留一法（LOO）
- K=n的K-fold，适合样本极少（n<30）
- 计算量大，n大时慎用

### 自助法（Bootstrap）
- 有放回抽样B次，每次抽n个样本构建训练集
- 适用于样本量小且需要估计置信区间的场景

## 四、灵敏度分析

### 局部灵敏度分析

**原理**：逐个参数变化，其他参数固定，观察输出的变化幅度。

**步骤**：
1. 选定要分析的参数（通常是模型假设中的关键参数或具有不确定性的参数）
2. 确定扰动范围（±5%, ±10%, ±20%）
3. 每次只改变一个参数
4. 记录输出的变化
5. 绘制灵敏度曲线或龙卷风图

**灵敏度系数**：
- S = ΔOutput / ΔInput（输出变化量/输入变化量）
- 或 S = (ΔOutput/Output₀) / (ΔInput/Input₀)（弹性系数）

**判断标准**：
- |S| < 0.2：不敏感，模型对参数变化稳健
- 0.2 ≤ |S| ≤ 1：中度敏感，需关注
- |S| > 1：高度敏感，需精确获取该参数或注明模型局限性

**龙卷风图绘制要点**：
- 横向条形图，中间为基准值
- 参数从上到下按影响大小排列
- 直观展示各参数的相对重要程度

### 全局灵敏度分析

**原理**：同时考虑所有参数的不确定性（包括交互效应）。

**Sobol方法**：
- 一阶Sobol指数 S₁：参数Xᵢ单独对输出方差的贡献比例
- 总效应Sobol指数 S_T：Xᵢ及其与其他参数交互作用的总贡献
- 若S₁≈S_T，则Xᵢ与其他参数几乎无交互
- 若S_T-S₁较大，则Xᵢ主要通过交互作用影响输出

**Morris法**：
- 计算每个参数的"基本效应"（elementary effect）
- 给出均值μ（参数的重要程度）和标准差σ（非线性/交互程度）
- 适合参数较多时的初步筛选

### 灵敏度分析的书写规范

竞赛论文中的灵敏度分析章节应包含：
```
1. 分析目的说明
2. 选择的参数和扰动范围（并说明选择这些参数的理由）
3. 使用方法名称（Morris法/Sobol法/单参数扰动）
4. 结果展示（表+图）
5. 结论：哪个参数最敏感？模型是否稳健？
6. 对实际问题的启示（如：需精确测量某参数 / 模型对某条件不敏感可放宽要求）
```

## 五、模型对比与选择

### 对比实验设计
- **控制变量**：除模型不同外，数据、预处理、评估标准完全一致
- **多维度对比**：精度、速度、可解释性、鲁棒性
- **统计显著性**：多个模型的对比应做统计检验（如McNemar检验、配对t检验）

### 对比示例：预测问题
| 模型 | RMSE | MAE | R² | 训练时间(s) |
|------|------|-----|----|------------|
| ARIMA | 12.34 | 9.87 | 0.85 | 2.3 |
| SVR | 8.56 | 6.21 | 0.91 | 15.7 |
| XGBoost | 5.43 | 3.92 | 0.96 | 45.2 |
| LSTM | 4.89 | 3.45 | 0.97 | 180.5 |

需给出选择某种模型的理由（精度优先选LSTM，实时性优先选ARIMA）。

## 六、模型合理性检验

### 定性检验方法
1. **边界检验**：输入极端值，输出是否合理
   - 例：需求预测中，价格为无穷大时需求应趋于0
2. **趋势检验**：输出趋势是否与理论预期一致
   - 例：传染病模型中总感染人数不应超过总人口
3. **量纲检验**：检查方程两边量纲是否一致
4. **守恒检验**：物理系统中的质量/能量/动量守恒是否满足

### 定量检验方法
1. **与基准模型对比**：与简单模型（如均值预测、线性回归）对比，确保复杂模型确实更好
2. **与文献结果对比**：相似场景下与其他论文的结果对比
3. **假设后验检验**：建模假设在结果中是否成立（如正态性假设）

## 七、不确定性量化

### 蒙特卡洛不确定性传播
当输入参数有不确定性（如测量误差）时：
1. 定义每个不确定参数的分布（如正态分布N(μ, σ²)）
2. 从各分布中抽样N次（N=10000+）
3. 每次运行模型，得到N个输出值
4. 统计输出的分布（均值、标准差、置信区间）
5. 输出形式：X = μ̂ ± 1.96σ̂（95%置信区间）

### 置信区间
- 95%置信区间：point estimate ± 1.96 × SE
- Bootstrap置信区间：适用

## 八、报告中的展示规范

- 所有误差数据保留2-3位有效数字
- 对比实验用粗体标记最优值
- 灵敏度曲线图包含基准线（红线虚线标记扰动=0时的原始值）
- 表格上标、下标注明检验条件
- 灵敏度分析注明扰动范围和步长
