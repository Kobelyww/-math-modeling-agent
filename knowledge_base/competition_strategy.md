# 数学建模竞赛策略与实战经验

## 一、选题策略

### 选题原则
1. **数据可获得性优先**：优先选择数据容易获取或有明确数据来源的题目
2. **方法匹配度**：选择团队擅长的建模方法（优化类/统计类/微分方程类）
3. **区分度考量**：避开太热门导致大量雷同方案，也避开太冷门导致参考资料太少
4. **时间可行性**：估算每个问题的难度和工作量，确保3天内可完成
5. **评委视角**：结果能出直观漂亮的图表、有清晰数值答案的题目通常得分更高

### 题型特征识别
| 题目类型 | 关键词 | 常见方法 |
|----------|--------|----------|
| 优化型 | 最短/最小/最大/最优方案 | 规划模型、遗传算法、模拟退火 |
| 预测型 | 预测/趋势/估计 | 时间序列、回归、机器学习、灰色预测 |
| 评价型 | 评价/排名/分级/综合 | AHP、TOPSIS、模糊综合评价、熵权法 |
| 机理型 | 机理/规律/物理过程 | 微分方程、元胞自动机、系统动力学 |
| 数据分析型 | 大数据/挖掘/识别 | 聚类、分类、降维、关联规则 |

### 选题的避坑指南
- 题目有大量专业术语和概念 → 理解成本高，慎选
- 题目需要特定领域知识（如金融衍生品定价）→ 缺乏背景知识慎选
- 题目只有定性描述没有数据 → 需自行找数据，增加工作量
- 看起来简单但实际需要超大数据处理能力 → 硬件限制

## 二、团队分工

### 三人团队的标准配置
1. **建模手**（最重要）：负责模型建立、推导、灵敏度分析
   - 需要：数学功底强，思维严谨，能从实际问题抽象数学模型
2. **编程手**：负责模型实现、数值实验、数据可视化
   - 需要：Python/MATLAB熟练，能快速将数学模型转化为代码
3. **写作手**：负责论文撰写、排版、图表制作
   - 需要：文字表达能力强，排版细致，有竞赛论文写作经验

### 分工协作模式
- **串行模式**（推荐新手）：
  建模手建好模型 → 编程手实现求解 → 写作手撰写论文
  优点：流程清晰，减少混乱
  缺点：下游等待上游，效率较低

- **并行-串行混合模式**（推荐有经验团队）：
  Day1：三人一起讨论选题和理解问题
  Day1-2：建模手和编程手紧密配合（边建模边实现），写作手同步开始写问题重述、假设、符号说明
  Day2-3：编程手出最终结果后，写作手重点完成摘要和模型求解章节
  优点：并行推进，效率高
  要点：需要频繁沟通，确保建模和写作信息同步

### 每日时间表示例
```
Day 1:
  08:00-10:00  选题讨论（3人一起）
  10:00-12:00  文献调研，问题分析
  13:00-18:00  模型建立（建模手+编程手），论文框架搭建（写作手）
  19:00-23:00  第一版模型run出初步结果

Day 2:
  08:00-12:00  模型优化+补充建模（建模手），代码完善+调参（编程手），正文撰写（写作手）
  13:00-18:00  灵敏度分析（建模手+编程手），论文核心章节完成（写作手）
  19:00-23:00  全部模型结果定稿，论文初稿完成

Day 3:
  08:00-12:00  论文修改+摘要精修（3人一起）
  13:00-16:00  排版+格式检查+查重降重
  16:00-18:00  最终通读+提交
```

## 三、模型选择的策略

### 模型复杂度权衡
- **简单模型优先**：能用线性回归解决的不要上深度学习（评委更看重建模思路而非工具复杂度）
- **创新在组合**：单个模型都常见，但**新颖的组合方式**是加分点
  - 例：先用K-means聚类分群，再对每群建ARIMA预测 + 熵权-TOPSIS选最优方案
- **模型序列**：先建基础模型 → 分析不足 → 引入改进模型 → 对比证明改进有效

### 体现建模深度的技巧
1. 不只说"我用了AHP"，要说明为什么用AHP而不是别的评价方法
2. 不只给出结果，要解释结果背后的直觉
3. 给出多种方法的结果对比
4. 对关键假设做灵敏度分析（几乎是必备环节）
5. 模型错误/局限主动讨论（展现学术诚实和分析能力）

## 四、常见题型应对模板

### 优化类题目（如：资源分配、路径规划、调度问题）
```
1. 明确决策变量：我们要决定什么？
2. 确定目标函数：最小化成本？最大化效率？
3. 列出约束条件：资源限制、物理限制、逻辑约束
4. 选择求解方法：精确解（LP/MILP）还是近似解（启发式）？
5. 验证与灵敏度：改变关键参数，结论是否改变？
```

### 预测类题目（如：销量预测、交通流预测）
```
1. 数据探索：趋势、周期、异常值
2. 特征工程：滞后期、滑动窗口、日期特征
3. 基准模型：简单方法（均值/线性回归）作为对比基线
4. 高级模型：根据数据特点选择（小样本→灰色预测，时序→ARIMA/LSTM）
5. 模型对比：精度+稳健性多维度对比
6. 预测区间：给出置信区间而非点预测
```

### 评价类题目（如：供应商评选、城市宜居度排名）
```
1. 建立指标体系（全面、独立、可量化）
2. 数据预处理（正向化、标准化、缺失值处理）
3. 确定权重（主观：AHP，客观：熵权法，组合：主客观综合）
4. 综合评价（TOPSIS/模糊综合评价/灰色关联分析）
5. 结果分析：排名合理性检验、指标贡献度分析
```

## 五、获奖论文的共同特点

分析历年国奖论文，总结以下共性特征：
1. **摘要极其精炼**：800-1000字包含所有核心内容和关键数值
2. **图表制作精美**：配色统一、字号清晰、加粗标注重点
3. **模型有层次**：不是单一模型，而是递进式的模型体系
4. **检 验全面**：有灵敏度分析、模型对比、误差分析
5. **排版整洁**：三线表、公式编号、段落间距均匀
6. **有超出题目要求的扩展**：自主提出改进或推广
7. **结果呈现直观**：关键数字用粗体/方框标注，让评委一眼看到
8. **参考文献规范且数量适中**：10-20篇，中英文混合

## 六、实用资源清单

### 常用Python库
- 数据处理：pandas, numpy
- 优化求解：scipy.optimize, pulp, cvxopt, gurobipy
- 机器学习：scikit-learn, xgboost, lightgbm
- 深度学习：pytorch, tensorflow（仅复杂问题需要）
- 时间序列：statsmodels, prophet
- 可视化：matplotlib, seaborn, plotly
- 图论：networkx
- 统计分析：scipy.stats

### 常用软件与工具
- LaTeX或Word（论文排版）
- Draw.io或Visio（流程图）
- MATLAB（如果团队更熟悉）
- Excel（快速数据探索和处理）

### 数据处理注意事项
- 缺失值：删除（缺失率>50%）/ 填充（均值/中位数/插值/KNN填充）
- 异常值：箱线图检测（IQR方法）、3σ原则、孤立森林
- 数据变换：标准化（Z-score）/ 归一化（MinMax）/ Box-Cox变换
- 数据不平衡：SMOTE过采样、欠采样、代价敏感学习
