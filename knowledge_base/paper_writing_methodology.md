# 数学建模竞赛论文写作方法论

## 一、论文整体结构

国赛/美赛论文的标准结构（按顺序）：
1. **摘要** — 最重要的部分，评委最先阅读
2. **问题重述与分析**
3. **模型假设与符号说明**
4. **模型建立与求解**（核心章节）
5. **模型检验与灵敏度分析**
6. **模型评价与改进**
7. **参考文献**
8. **附录**（代码、详细数据）

## 二、摘要撰写要点

### 摘要的重要性
- 摘要占总分的比重最大（国赛约占40%，美赛约占30%）
- 多数评委先看摘要再决定是否继续阅读正文
- 必须**独立成文**：即使不读正文也能理解核心内容

### 摘要的黄金结构（约800-1000字）
1. **背景一句**：简要说明问题背景（1-2句）
2. **问题一**：
   - 使用什么方法/模型（如：建立基于XGBoost的预测模型）
   - 得到什么结果（给出关键数值）
   - 结果说明什么
3. **问题二**：（同上结构）
4. **问题三**：（同上结构）
5. **创新点**：本文的主要创新与亮点（2-3条）
6. **关键词**：5-8个关键词

### 摘要常见错误
- 过于笼统（"使用了综合评价方法"→应写"熵权-TOPSIS综合评价模型"）
- 缺少数值结果（只描述过程，不给具体数字）
- 背景过长（摘要用1/3篇幅写背景=浪费）
- 没有提及创新点

## 三、问题重述与分析

### 写作要点
- **不要直接复制题目**，用自己的语言重新组织
- 明确输入是什么、输出是什么、约束条件是什么
- 画流程图展示整体解题思路框架
- 分析每个子问题的难点和关键挑战

### 典型句式
- "问题X本质上是一个xxx优化问题（或 xxx分类/预测/评价问题）"
- "该问题的核心难点在于...（数据量少/多目标冲突/约束复杂/不确定性高）"
- "针对这一特点，本文采用xxx方法，其优势在于..."

## 四、模型假设与符号说明

### 模型假设的原则
1. **合理性**：假设有现实依据，不是随意简化
2. **必要性**：每个假设都是为了简化模型或使求解可行
3. **敏感性**：关键假设需在后续做灵敏度分析

### 假设分类
| 类别 | 示例 |
|------|------|
| 简化假设 | "假设各充电站服务能力相同" |
| 理想化假设 | "假设不考虑极端天气影响" |
| 数据假设 | "假设数据服从正态分布" |
| 边界假设 | "仅考虑2024-2030年的时间范围" |

### 符号说明
- 使用三线表（booktabs风格）统一列出
- 符号按字母顺序或出现顺序排列
- 每个符号注明：符号、含义、单位
- 全文符号一致，不重复定义

## 五、模型建立与求解

### 建模的三步法
1. **问题抽象**：将实际问题转化为数学语言（变量、约束、目标）
2. **模型选择**：说明为什么选择这个模型（引用文献或对比分析）
3. **模型建立**：写出完整的数学表达式

### 模型描述的层次
- **文字描述**：先通俗解释模型思路（让非专业评审也能理解）
- **数学公式**：给出严格的数学表达式
- **算法流程**：画流程图或伪代码展示求解步骤
- **数值结果**：展示具体计算结果（表、图）

### 求解部分的写作
- 说明使用了什么算法 / 什么软件（Python scipy / gurobi / 自编算法）
- 如果使用启发式算法，需解释参数选择和收敛性
- 关键中间结果用表格展示
- 最终结果用粗体或着重标记

### 多模型对比的写作方法
- 并列展示2-3种方法的结果
- 分析各方法的优劣（精度 / 计算时间 / 可解释性）
- 最终选择何种方法及其理由

## 六、模型检验与灵敏度分析

### 模型检验的方法
1. **数值检验**：与实际数据或标准答案对比，计算误差指标
   - 回归问题：MSE, MAE, R², MAPE
   - 分类问题：准确率, 精确率, 召回率, F1-score, AUC
2. **稳定性检验**：轻微扰动输入，观察输出变化
3. **交叉验证**：K-fold交叉验证
4. **极端情况检验**：对边界/极端输入验证模型行为

### 灵敏度分析的写法
- **方法选择**：局部灵敏度（Morris法）、全局灵敏度（Sobol法）
- **书写要素**：
  - 扰动哪些参数？（选择影响较大的关键参数，±5%, ±10%, ±20%）
  - 观察哪些指标的变化？
  - 用图表展示（灵敏度曲线、龙卷风图、热力图）
  - 分析哪个参数最敏感 → 对实际应用的启示
- **结论模板**："参数α在±20%内波动时，目标函数仅变化<-3%, 5%>，说明模型对α不敏感，结论稳健"

## 七、模型评价与改进

### 模型评价（优缺点）
- **优点**（3-5条）：针对性强，不泛泛写"计算效率高"
  - 正确示范："采用LSTM捕捉时序依赖，MAPE低于5%，优于ARIMA的12%"
  - 错误示范："我们的模型精度高、速度快"（需有量化支撑）
- **缺点**（2-3条）：诚实但有建设性
  - 正确示范："受限于数据量，模型在极端场景下的预测能力有待验证"
  - 错误示范：不要写致命缺陷

### 模型改进/推广
- 提出2-3个可行的改进方向（不是空想，是有依据的建议）
- 说明改进需要什么额外条件（数据、算法、计算资源）
- 模型在其他领域的潜在应用

## 八、写作技巧与常见问题

### 图表规范
- 表标题在上方、图标题在下方
- 表格使用三线表（只有顶线、栏目线、底线）
- 图片清晰（矢量图优先，确保打印后还清晰）
- 折线图：线条≤4条，否则分图
- 柱状图：类别标签过长时旋转45°
- 每个图表在正文中必须有引用

### 公式规范
- 重要公式单独成行并编号（使用括号编号如(1), (2)）
- 行内公式与正文流畅衔接
- 符号首次出现必须解释
- 使用专业排版（LaTeX或Word公式编辑器）

### 参考文献格式
- 引用最新文献（最近3-5年为主）
- 中英文文献均引用（显示文献调研充分）
- 参考文献数量：10-20篇
- 格式一致（GB/T 7714 或 IEEE）

### 写作时间分配建议（以3天竞赛为例）
| 时间段 | 任务 |
|--------|------|
| Day 1 上午 | 选题 + 理解问题 + 查找文献 |
| Day 1 下午 | 模型建立（所有题目） |
| Day 1 晚上 | 编程实现 + 初步结果 |
| Day 2 上午 | 调参 + 模型优化 + 灵敏度分析 |
| Day 2 下午 | 开始写论文正文（重点：模型与求解） |
| Day 2 晚上 | 完成论文初稿 |
| Day 3 上午 | 论文修改 + 摘要精修 |
| Day 3 下午 | 排版检查 + 最终定稿提交 |

### 加分技巧
- 给出超出题目要求的新思路或扩展分析
- 创新性地结合多种方法（如"基于熵权-TOPSIS和K-means的两阶段评价模型"）
- 设计直观有效的图表展示核心结果
- 摘要中体现量化结果和专业术语
